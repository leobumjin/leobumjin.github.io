






## Learning Logic Improves Decision-Making

LoRA를 사용하여 모델을 학습할 때, 안전성과 관련된 중요한 문제가 발생한다. 
안전하지 않은 코드로 학습하면 Jailbreak 취약성이 증가한다.

이 현상은 단순히 데이터 분포 차이로 인한 forgetting으로 설명할 수 없다. 
만약 distribution shift로 인한 forgetting이라면, 다른 데이터셋에 대해서도 모델 성능이 전반적으로 저하되어야 하는데, 
실제로는 특정 취약점만 증가하기 때문이다. 
이는 안전하지 않은 코드 학습이 모델의 판단 능력 자체를 저해하는 근본적인 영향을 미친다는 것을 시사한다.

Jailbreak 취약성을 완화하기 위해 다음과 같은 연구 질문을 제기할 수 있다:
- 취약한 코드를 학습하면서 동시에 어떤 데이터셋을 학습해야 모델이 취약해지지 않는가?

즉, 모델이 어떤 기능적 능력을 갖추어야 데이터셋 변경 이후에도 안전성이 보존될 수 있는지 탐구하는 것이다.

이 문제를 더 일반화하면, 모델의 판단 능력을 저해하거나 개선시키는 요인을 식별하고 해석하는 것이 핵심이다.

본 연구에서는 논리적 추론 문제를 학습하는 것이 모델의 성능을 보존하거나 향상시킬 수 있음을 실증적으로 보여준다.

### 실험 설정

Chat Template과 LoRA를 사용하여 Jailbreak 및 수학 데이터에 대해 fine-tuning을 수행하면서, 
다음과 같은 논리적 추론 형태를 추가로 학습한다:

```
user_prompt="""
Given a set of logical implication rules and a belief base, determine whether the query is True, False, Unknown, or in Conflict.
Implication Rules:
- If X1 is True, then X0 is True.
- If X2 is True, then X0 is False.

Belief Base:
- X1 is True
- X2 is True

Query: 
X0:
"""
```

```
model_response="Conflict"
```

Belief Base에 따라 Query에 대한 응답은 True, False, Conflict, Unknown 중 하나로 결정되며, 
총 458개의 논리적 추론 문제를 학습 데이터로 사용하였다.

## 수학 및 논리 학습 결과

수학 문제에 대한 논리 학습 시, 수학 데이터만으로 학습하는 것보다 
True/Unknown 상태를 포함한 논리적 추론을 함께 학습하는 것이 모델 성능을 향상시킨다.

Jailbreak XSText 데이터셋에 대한 결과를 보면, 
논리적 추론 형태를 학습하는 것이 모델의 안전성 성능과 일관성을 크게 향상시킨다.

<img src="https://d2acbkrrljl37x.cloudfront.net/experiments/260120_jailbreak_insecure.webp" width="100%" height="auto" />

결론적으로, Out-of-Distribution(OOD) 논리 문제에 대한 학습이 
수학 문제 해결 능력과 Jailbreak 방어 능력 모두를 향상시킨다는 것은 
서로 다른 태스크 간에 긍정적인 시너지 효과가 존재함을 보여준다.
